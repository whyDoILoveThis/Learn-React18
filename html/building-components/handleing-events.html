Click Event on a Component

The Goal Here..

We want to click each item, and have it printed to the console.

Each element in react has a prop called onClick.

This will do for now..

If you get err just refresh.

So we get the clicked message, but what if we want the name of the item?

Just log the item.

onClick={() => console.log(item)}

Index

We can also add a second parameter to access the index.

Event Parameter

Lets add an event parameter and log it to the console.

onClick={( event ) => console.log(event)}

What you get is a built in component of react.

Not all browsers handle events the same way, so react created this to solve that problem.

Separate functions

If your event handles get complex you should move them to a separate function.

Why do we have this warning??

implicitly has 'any' type.

It means Typescript doesn't know its type.

Is it a number? A bool? A string? What is it?

So if we use a dot on the event object we dont see any propertys.

We need to specify the type of the parameter.

Parameter Type

Using the function

Take note that we are not CALLING the function with ();

We are just passing it as a reference. We are telling react to call it when an item is clicked.

so calling the function gets done later, at runtime.
