Passing Data Via Props

Why we need props

So, we've got a list of cities.

But, what if we had names or colors?

We don't want to completely remake this component for each kind of list.

We want to make this component reuseable. This is where we use propertys(props).

What are props

Props are the inputs to our components.

It's like we are creating our own custom html attributes for our custom components.

We kind of make a spot ready for a variable to be inserted, and the value of that variable is used as the data of the component to be displayed.

Typescript Interface

We need to decide the shape of the inputs to our ListGroup component.

We are going to want to be able to pass an object with 2 propertys.

'items', which is an array, and 'heading', which is a string.

To do that we use a Typescript feature called an 'interface'. With this we can define the 'shape' or the 'interface' of an object.
